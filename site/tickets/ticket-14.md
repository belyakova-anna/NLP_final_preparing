## Q2. In-context learning techniques: Generated Knowledge Prompting и Self-Consistency

In-context learning — это группа техник, при которых **поведение LLM меняется только за счёт промпта** (без обновления весов). Few-shot prompting и Chain-of-Thought — базовые из них; **Generated Knowledge Prompting** и **Self-Consistency** — более продвинутые.

### Generated Knowledge Prompting

**Generated Knowledge Prompting** (Liu et al., ACL 2022) — техника двухпроходного промптинга: сначала модель сама **генерирует релевантные знания** по теме вопроса, потом эти знания подаются обратно в модель вместе с исходным вопросом, чтобы помочь ей дать правильный ответ.

#### Мотивация

LLM знает многое, но не всегда **извлекает** нужное знание спонтанно при ответе. Прямой вопрос «Является ли X истиной?» может получить неверный ответ, потому что модель не «вспомнила» нужный факт. Generated Knowledge Prompting заставляет модель **сначала развернуть знание**, потом им воспользоваться — что-то вроде явного RAG-а из собственных весов.

#### Алгоритм

1. **Knowledge generation step**: даётся промпт «Generate some knowledge about [topic]:», few-shot демонстрации содержат пары `(вопрос → ключевые факты)`. Модель генерирует $K$ кандидатов знаний $k_1, \ldots, k_K$ (можно с разными температурами).

2. **Knowledge integration step**: для каждого $k_i$ формируется новый промпт `Knowledge: {k_i}\nQuestion: {q}\nAnswer:`. Модель отвечает на вопрос, **обусловленная** на сгенерированное знание.

3. **Aggregation**: вероятности правильного ответа усредняются (или берётся most-confident прогноз) по разным $k_i$.

Псевдокод:

```
knowledges = [LM("Generate knowledge: " + few_shot + q) for _ in range(K)]
answers = [LM(f"Knowledge: {k}\nQ: {q}\nA:") for k in knowledges]
final = aggregate(answers)  # voting или max-confidence
```

#### Эффект и применимость

В оригинальной статье на бенчмарках по common-sense reasoning (NumerSense, CSQA, CSQA2, QASC) Generated Knowledge Prompting улучшает качество на 5–10 п.п. без какого-либо обучения. Особенно полезно там, где правильный ответ требует явного знания (например, физических или социальных норм). Метод хорошо комбинируется с CoT: сначала knowledge, потом step-by-step reasoning.

### Self-Consistency

**Self-Consistency** (Wang et al., ICLR 2023, «Self-Consistency Improves Chain of Thought Reasoning in Language Models») — техника декодирования, существенно усиливающая Chain-of-Thought: вместо одной greedy CoT-цепочки **семплируется $N$ разных** цепочек рассуждения, и финальный ответ выбирается **голосованием большинства** среди их final answers.

#### Мотивация

Сложные reasoning-задачи имеют **много правильных путей рассуждения**, но конкретный путь может содержать ошибку из-за стохастичности генерации или плохого выбора шага. Если правильных рассуждений больше, чем неправильных, и они **сходятся к одному ответу**, голосование исключит шум.

#### Алгоритм

1. Используем CoT-промпт (few-shot или zero-shot «Let's think step by step»);
2. Семплируем $N$ ответов с **температурой $T > 0$** (например, $T = 0.7$, top-p 0.95) — каждый ответ содержит рассуждение и финальный numerical/textual answer;
3. Извлекаем final answer из каждой цепочки (по парсингу «The answer is X» или regex);
4. Применяем **majority voting**:
$$\hat{y} = \arg\max_y \sum_{i=1}^{N} \mathbb{1}[\text{answer}_i = y].$$

#### Иллюстрация

Задача: «Если в коробке 5 яблок и я съел 2, потом купил ещё 3, сколько яблок?»

CoT-цепочки:
1. «5−2=3, 3+3=6 → 6» ✓
2. «5−2=3, 3+3=6 → 6» ✓
3. «5+3=8, 8−2=6 → 6» ✓
4. «5−2=3, 3+3=7 → 7» ✗ (арифметическая ошибка)
5. «5−2=3, 3+3=6 → 6» ✓

Голосование: «6» — 4 голоса, «7» — 1 → ответ 6.

#### Условия и эффект

- **Требует CoT** — на коротких прямых ответах не помогает (нет вариативности);
- **Требует семплинга** — greedy / temperature 0 не имеет смысла (всегда одно и то же);
- **Цена** — в $N$ раз больше токенов (обычно $N = 5$–$40$);
- **Прирост**: на GSM8K (математическое reasoning) Self-Consistency + CoT дают +17 п.п. над голым CoT для PaLM 540B; на MultiArith — +7 п.п.

#### Связь с другими техниками

- **Tree-of-Thoughts** (Yao et al., 2023) — обобщение: вместо плоского голосования строится дерево промежуточных мыслей с эвристикой выбора лучших ветвей;
- **Best-of-N + reward model** — если есть RM, выбираем не голосованием, а максимумом награды;
- **MCTS-like search** — формальный поиск по дереву рассуждений с rollout'ами;
- **Verification chains** — отдельная LLM-голова проверяет каждое рассуждение, голосование взвешивается.

### Сравнение

| Метод | Что делает | Когда полезен |
|---|---|---|
| Few-shot | Демонстрации в промпте | Любая задача, где есть формат |
| CoT | Промежуточные шаги рассуждения | Reasoning, math, multi-step |
| Generated Knowledge | Сгенерировать знания, потом ответить | Common-sense, factual |
| Self-Consistency | $N$ CoT-семплов + голосование | Reasoning, где есть шум в путях |
| ToT / GoT | Дерево/граф рассуждений с поиском | Очень сложные task'и |
